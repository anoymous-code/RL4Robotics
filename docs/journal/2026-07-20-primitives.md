# 2026-07-20 (深夜) · 原语分解：学习只放在刀刃上，全流程回到 65~80%

## 架构：把"学习"限制在非学不可的地方

端到端 ACT 用一个网络吃下全部 55 秒流程，但[分辨率实验](2026-07-20-hires.md)
之后看得很清楚：流程的绝大部分（运送、转体、对位）本来就是脚本 IK 的
强项，真正需要学习的只有 4 个接触密集窗口。于是重构为
**原语分解 + 经典控制衔接**的混合架构：

```mermaid
graph LR
    subgraph 脚本衔接段（IK 高精度，永远走脚本）
        A["接近药板"] --> B["提板转体"] --> C["接近目标格"]
        C --> D["运送到盒 B"] --> E["撤臂/对槽"]
    end
    A -.->|窗口| p1["p1 抓板<br/>(学习)"]
    C -.->|窗口| p2["p2 撕剪<br/>(学习)"]
    D -.->|窗口| p3["p3 投放<br/>(学习)"]
    E -.->|窗口| p4["p4 插回<br/>(学习)"]
```

每个原语是**单臂、单腕相机、7 维动作**的小 ACT 策略（480×640 观测，
特征网格 15×20 不池化——单相机付得起，空间细节全保留）：

- 训练数据从专家演示的对应窗口切片（入口状态池：跑全流程专家，
  在每个原语入口打状态快照 + 出口谓词回填可行性标签）；
- 评测口径 = "衔接段把臂送到入口（含 4mm 到位抖动）后，原语能否
  完成"（出口谓词真值判定）；
- 编排器（`run_primitives.py`）支持逐原语上线 + **原语级重试**：
  出口谓词不满足 → 张爪撤回入口位形再来一次——这是端到端策略
  给不了的结构化容错。

## 结果（严格物理，双指规范撕剪口径）

单原语（入口池可行条目，n=40）：撕剪原语 p2 = **62%**——与端到端
act_hires 的撕剪成功率一致（60%），符合预期：p2 就是毫米级瓶颈本身，
换架构不会凭空提精度。

编排器全流程（4 原语全学习 + 每原语重试 1 次，n=20 × 两轮）：

| 指标 | 端到端 act_hires | **原语分解 + 重试** |
|---|---|---|
| 撕剪入盒 B | 60% | **85~95%** |
| 全流程 | 45% | **65~80%** |
| p1 抓板 / p3 投放 / p4 插回 | —（不可分项） | 95~100% / 85~95% / 75~85% |

提升不来自"学得更准"，而来自**架构红利**：

1. **重试**：p2 单发 62%，重试一次后编排器里 85~95%——接触失败
   大多是"这次没夹准"，退回入口再来一次就好。端到端策略没有
   "入口"概念，失败即滑向分布外；
2. **衔接段零误差**：运送/转体交回脚本 IK，端到端策略在这些段落
   积累的漂移（虽小但会级联进抓取窗口）被清零；
3. **失败隔离**：p2 失败时张爪撤回，不会夹着未撕断的格拖走整板
   ——终局判定仍可能救回部分分数。

## 踩坑：mujoco Renderer 的连坐 close

第一版编排器视频全流程黑屏（只有前 5% 正常）。原因：每个原语窗口
的 `PrimitiveSession` 自建渲染器并在窗口结束时 `close()`——mujoco
的 `Renderer.close()` 会销毁共享 GL 上下文，把同 episode 还活着的
录像渲染器一起杀掉。修复：episode 级共享一个策略观测渲染器
（外部注入，持有者管生命周期），录像渲染器与其隔离。

## 定位与下一步

- 这条线验证了**混合架构的工程价值**：同样的 62% 原语精度，套上
  "经典衔接 + 结构化重试"的外骨骼后全流程翻身。产品视角它也更
  可审计（每个原语有独立成功率、独立视频、独立重试预算）；
- 真正的精度提升（p2 62% → 90%+）仍要走已列的路：抓取区裁剪放大 /
  腕相机 720p / BC+RL 微调——现在这些都可以**只对 p2 一个原语做**，
  实验成本从全流程级降到原语级（快 4 倍）。

## 做了什么

### 1. 场景参数化 + 域随机化（`SceneCfg` / `sample_cfg`）

盒 A/盒 B 位置、停车位姿、目标格不再是写死的常量，而是每个 episode 采样的
`SceneCfg`。XML 按 cfg 程序化生成——这就是把"世界写进代码"改成"世界作为参数"，
是所有学习方法的地基。`level` 参数可缩放扰动幅度（课程学习备用）。

### 2. 机载观测相机（head_cam）

之前的主视角是"上帝相机"（`targetbody` 自动跟踪药板），真机没有这种东西。
新加的 `head_cam` 固定在机器人立柱顶部俯视桌面，与两个腕相机构成三路观测——
**与 Mobile ALOHA 真机的相机配置同构**，策略学到的观测依赖可以直接搬到真机。

### 3. 脚本专家 = 数据生成器

`FullDemo` 改造：吃 `SceneCfg`、可跳过导航段（episode 从停稳后开始，学习聚焦操作）、
目标格可配、挂 `DemoRecorder` 钩子——每个控制周期记录（观测，动作=当前伺服目标）。
一条命令自动采集：

```powershell
..\..\.venv\Scripts\python.exe collect_demos.py --n 50 --seed 0
```

这正是第二课说的"脚本专家是最便宜的数据印钞机"（MimicGen 思路）：
不需要人遥操作，睡一觉起来几百条演示就采好了。

### 4. Gymnasium 环境（`pill_env.py`）

标准 `reset()/step()` 接口：reset 即域随机化重建场景，动作空间 14 维
（双臂关节 + 爪，ALOHA/ACT 标准布局），观测 = `qpos` + 三路 240x320 图像。
稀疏奖励（目标格入盒 B、剩板回槽）留给以后的 RL；`privileged()` 暴露真值位姿，
是 RL 教师-学生蒸馏的接口。烟测通过。

## 踩坑与解决

1. **上帝视角不能当观测**：`follow_pack` 相机 mode="targetbody" 会自动跟踪药板——
   等于把物体位置泄漏给了策略。观测相机必须是"真机装得出来"的固定机位。
2. **奖励判定的初始态陷阱**："剩板在槽中"在 episode 开始时天然成立，直接给分
   会奖励"什么都不做"。改为"目标格入盒 B 之后"回槽才算数。
3. **数据体积**：50 Hz × 3 相机 × 1900 步，原始图像一条就要 1.3 GB。
   JPEG（质量 60）后 ~8 KB/帧，一条 50 MB，可以接受；`demos/` 已加入 `.gitignore`。

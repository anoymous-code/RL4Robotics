# 2026-07-20 (下) · 眼镜起作用了：分辨率 ×2，规范撕剪 15% → 60%

## 检验的假说

[上一篇](2026-07-20-precision.md)把瓶颈钉在"从像素回归毫米"：
240×320 腕相机里 2.4 mm 板缘只占 5~8 像素，±2 mm 的抓取定位要求
淹没在回归噪声里。三种数据配方（更多演示、教师蒸馏、短开环）都
拉不动 15%，指向的都是同一个论断——**信息不够，不是学得不好**。

假说的直接检验：观测分辨率 ×2（480×640），同样的模型、同样的
专家、同样的评测协议，看成功率动不动。

## 实现：像素 ×4，token 数不变

分辨率 ×4 像素若直接进 Transformer，token 从 240 涨到 900，
注意力成本 ×14——训练时间不可接受。取巧的地方：

```
480×640 → ResNet18（高分辨率卷积，细节进通道）→ 15×20 特征
        → 自适应平均池化 8×10 → token 数与位置编码完全不变
```

毫米级细节在 backbone 的高分辨率卷积阶段就被编码进特征通道，
池化只压缩空间排列。模型代码总共加了三行，旧权重结构不变。
成本：采集渲染 ×3.5（240 条 ~1.7 h）、训练 FLOPs ×4（batch 16→8，
20k 步 ~2.8 h）、数据体积 ×2.4（33 GB，JPEG 质量 60→50）。

## 结果：4 倍

严格物理（双指夹持才可断裂），同一场景种子序列与物理参数序列：

| 规范撕剪 / 全流程 | act_strict (240×320) | **act_hires (480×640)** |
|---|---|---|
| 标称物理 | 15% / 10% | **60% / 45%** |
| 物理随机化 | 15% / 5% | **45% / 40%** |

而且每一次成功都是规范撕剪（敲断 0 次——严格物理下敲断在物理上
不可能成功）。专家采集成功率 72%（vs 低分辨率版 74%，专家读真值
坐标、不受分辨率影响，这也反向确认对照干净）。

## 分析

1. **信息瓶颈假说成立**。同样的模型容量、数据量、训练步数、专家，
   唯一变量是像素密度，规范撕剪 ×4。之前三轮"数据配方"实验的
   全部失败一并得到解释：往一个信息饥饿的系统里灌更多标签没有用。
2. **60% 还不是天花板**。剩余失败仍集中在抓取（板缘对准差 1~2 mm
   量级），进一步的路径此刻才真正轮到：
   - 视觉端继续加码：只把腕相机再升到 720p / 裁剪放大抓取区域；
   - **BC 初始化 + RL 微调**：60% 的基线意味着 RL 的探索有足够的
     成功信号可以放大——15% 时代这条路是死的（成功太稀疏），
     现在"ACT + RL"第一次有了地基；
   - 抓取段视觉伺服作为工程兜底。
3. **成本换精度的账本**：×4 计算换 ×4 成功率，这一轮是整个项目
   性价比最高的一次投入。也是最"无聊"的一次——没有新算法，
   只是把感知的地基打够厚。**很多"策略学不会"的问题其实是
   "策略看不见"的问题。**
